HEURISTICS APPLIED
Key Design Principles
UI/UX principles that separate the best food delivery apps from the rest.
⬛
Visual Hierarchy
Clear info priority — name, rating, price, ETA — guides faster decisions.
🔽
Progressive Disclosure
Show only what's needed each step. Menu → Item → Customise → Cart.
✅
Feedback & Affordance
Every tap should respond. Loading states and micro-animations confirm actions.
🛡️
Error Prevention
Confirmations before cancellation, address validation, clear minimum order alerts.
🔗
Consistency
Uniform icon language, button styles, and colour usage across all screens.
♿
Accessibility
AA contrast, 44×44px touch targets, screen reader labels, keyboard nav.

UX PROBLEMS IDENTIFIED
Systemic Flaws
Common UI/UX problems across food delivery apps that reduce trust and retention.
🍔
Dark Pattern Checkout
Surge pricing revealed only at checkout — not upfront — eroding user trust.
🔔
Notification Overload
Excessive promo notifications push users to disable all alerts, hurting retention.
📍
Address Friction
Re-entering delivery addresses is a key pain point — poor location memory UX.
⭐
Review Manipulation
Rating systems vulnerable to fake reviews, reducing the value of social proof.
💸
Hidden Fee Fatigue
Platform + packaging fees + GST shown only at cart — causing high abandonment.
🔍
Weak Search UX
Most apps lack semantic search — users must know exact names to find dishes.
♿
Accessibility Gaps
Poor contrast ratios and missing ARIA labels exclude users with disabilities.
